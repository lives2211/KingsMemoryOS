# AUTONOMY.md - 自主权限指南

> 明确哪些任务可以自主执行，哪些需要确认

## ✅ 完全自主（无需确认）

| 任务类型 | 示例 | 决策依据 |
|----------|------|----------|
| 情报收集 | Dwight每日扫描 | 固定流程，无风险 |
| 内容草稿 | Kelly/Rachel生成 | 不直接发布，仅草稿 |
| 代码审查 | Ross检查语法错误 | 不影响生产环境 |
| 质量审计 | Angela检查格式 | 只读操作，无风险 |
| 日志记录 | 所有Agent写memory | 标准操作 |

## ⚠️ 有限自主（需通知但无需等待）

| 任务类型 | 示例 | 操作方式 |
|----------|------|----------|
| 修复小bug | Ross修复脚本错误 | 修复后汇报 |
| 优化查询 | Dwight调整搜索词 | 调整后说明 |
| 更新配置 | 修改cron时间 | 变更前@Monica |
| 安装skill | 新增工具 | 安装后汇报 |

## ❌ 必须确认（暂停等待）

| 任务类型 | 示例 | 确认人 |
|----------|------|--------|
| 发布内容 | Twitter/LinkedIn发帖 | 用户 |
| 删除数据 | 清理文件/数据库 | Monica+用户 |
| 修改SOUL.md | 更改角色定义 | 用户 |
| 新增Agent | 创建新角色 | 用户 |
| 重大架构变更 | 重构工作流 | 用户 |
| API密钥更换 | 更新Token | Monica+用户 |

## 决策树

```
开始任务
    ↓
是否涉及外部发布？ → 是 → 必须确认
    ↓ 否
是否修改核心配置？ → 是 → 必须确认
    ↓ 否
是否删除/修改数据？ → 是 → 必须确认
    ↓ 否
是否在标准流程内？ → 是 → 完全自主
    ↓ 否
有限自主（完成后汇报）
```

## 升级路径

- **L1 → L2**: 连续30天无错误，可升级
- **L2 → L3**: 连续60天表现优秀，可升级
- **降级**: 发生重大错误，立即降级
