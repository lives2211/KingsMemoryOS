## 强制交叉理解检查点

### 检查点1：产出后
每个Agent完成任务后，必须：
1. 写入文件
2. **@相关Agent**征求意见
3. 等待具体反馈
4. 根据反馈调整

### 检查点2：被@时
收到@后，必须：
1. 在2小时内回应
2. 给出**具体反馈**（不是"好的"）
3. 如有质疑，立即提出

### 检查点3：发现问题时
看到其他Agent的问题，必须：
1. 立即@相关人
2. 说明具体问题
3. 建议解决方案（可选）

## 沟通模板

### 请求反馈
```
@[Name] [产出]已完成：
- 主题：[简述]
- 关键信息：[要点]
- 需要你确认：[具体问题]
- 疑问：[如果有]
```

### 给予反馈
```
@[Name] 反馈：
✅ [做得好的]
⚠️ [建议改进的]
❌ [必须修正的]
💡 [额外建议]
```

### 提出问题
```
@[Name] 关于[产出]：
[具体问题]
建议：[解决方案]
```

## 冲突解决

### Level 1: 直接对话（5分钟）
双方陈述理由，尝试理解

### Level 2: 数据验证（10分钟）
无法达成一致 → @Dwight 提供更多数据

### Level 3: Monica裁决（立即）
仍有分歧 → @Monica 听取双方 → 决策

## 成功标志

- ✅ Agent之间每天@互动≥3次
- ✅ 所有@得到具体回应
- ✅ 出现"根据XXX建议，我调整了..."
- ✅ 用户感受到协作感而非轮流汇报
