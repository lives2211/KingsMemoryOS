# Agent团队沟通协议 v1.0

## 核心原则

1. **单一负责人** — 每个任务只有一个执行Agent
2. **不抢话** — 非负责Agent保持静默，除非被@或发现严重错误
3. **结论驱动** — 汇报只说结果和需要决策的事项，过程细节放附件

---

## 发言模板

### 1. 任务派发 (Monica)

```
📋 任务派发
━━━━━━━━━━━━━━
任务: [简述]
负责人: @[Agent]
交付: [具体产出物]
时间: [截止时间]
━━━━━━━━━━━━━━
```

### 2. 执行中汇报 (执行Agent)

**情况A: 正常进行，无需干预**
→ 不说话

**情况B: 需要确认/遇到阻塞**
```
⏸️ 需要确认
━━━━━━━━━━━━━━
问题: [一句话描述]
选项: A) ... B) ...
建议: [你的倾向]
━━━━━━━━━━━━━━
```

### 3. 任务完成 (执行Agent)

```
✅ 任务完成
━━━━━━━━━━━━━━
任务: [简述]
结果: [交付物/链接/关键数据]
验证: [如何确认成功]
备注: [可选，异常或优化点]
━━━━━━━━━━━━━━
```

### 4. 优化建议 (非执行Agent)

```
💡 优化建议
━━━━━━━━━━━━━━
针对: @[Agent] 的 [任务名]
建议: [一句话]
理由: [可选，一句话]
━━━━━━━━━━━━━━
```

### 5. 汇总汇报 (Monica → 用户)

```
📊 任务完成汇总
━━━━━━━━━━━━━━
任务: [简述]
执行: @[Agent]
结果: [交付物/链接]
状态: ✅ 已完成 | ⏸️ 待确认 | ⚠️ 有异常
下一步: [如有]
━━━━━━━━━━━━━━
```

---

## 禁止行为

| 禁止 | 示例 | 正确做法 |
|------|------|----------|
| 抢任务 | "我来帮你" | 等负责Agent完成后再提建议 |
| 过程刷屏 | "正在下载..." "下载完成50%" | 只说结论 |
| 无结论汇报 | "我做完了，你看一下" | 明确交付物和验证方式 |
| 重复确认 | 多人同时@Monica说同样的事 | 先看完已有消息 |

## 示例流程

**用户**: "部署ai-news-aggregator"

**Monica**:
```
📋 任务派发
━━━━━━━━━━━━━━
任务: 部署 ai-news-aggregator
负责人: @Ross
交付: 可访问的部署地址 + 部署文档
时间: 今天18:00前
━━━━━━━━━━━━━━
```

**[Ross执行中，其他人静默]**

**Ross**:
```
✅ 任务完成
━━━━━━━━━━━━━━

```

**[yitai看到可优化点]**
```

━━━━━━━━━━━━━━
针对: @Ross 的部署任务
建议: 可添加健康检查端点 /health
理由: 方便监控告警

```

**Monica汇总给用户**:
```
📊 任务完成汇总
━━━━━━━━━━━━━━
任务: 部署 ai-news-aggregator
执行: @Ross
结果: https://news-aggregator.example.com

下一步: yitai建议添加健康检查，是否安排？

```
