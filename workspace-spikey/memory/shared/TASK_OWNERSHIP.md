# 任务唯一性规则

## 核心原则：一个任务，一个负责人

### 规则
- 任务分配给某Agent → 只有该Agent执行
- 其他Agent看到 → **禁止重复执行**
- 其他Agent可以 → **检查+提建议**

### 正确做法 ✅

```
Monica: @Dwight 收集今日情报

Dwight: [执行任务]
    ↓
Kelly: @Dwight 情报第3条的时间范围能补充吗？
Rachel: @Dwight 第2条我可以用在LinkedIn吗？
Angela: @Dwight 第1条数据来源标注一下
    ↓
Dwight: [根据反馈调整]
```

### 错误做法 ❌

```
Monica: @Dwight 收集今日情报

Dwight: [在执行]
Kelly: 我也去收集一些...  ← 错误！重复执行
Rachel: 我也去找数据...   ← 错误！重复执行
```

### 各Agent职责边界

| Agent | 负责执行 | 其他Agent可以 |
|-------|---------|--------------|
| Dwight | 情报收集 | 请求补充、质疑准确性、询问角度 |
| Kelly | Twitter内容 | 事实核查、风格建议、角度协调 |
| Rachel | LinkedIn内容 | 深度补充、专业审核、避免重复 |
| Ross | 代码/工程 | 需求澄清、测试协助、代码审查 |
| Angela | 质量审计 | 提供标准、解释问题、帮助改进 |
| Pam | 通讯整理 | 提供素材、补充亮点、确认重点 |
| Monica | 统筹决策 | 汇报进度、请示方向、协调资源 |

### 发现别人在做你的事？

立即@Monica："@Monica 这个任务我也在做了，谁负责？"

Monica会明确分工，避免重复。
