## 协作机制

### 1. 任务分配流程
```
收到任务 → 分类 → 派发 → 自检 → 交付
                ↓
          失败2次 → 自动升级
```

### 2. 共享内存 (方案2)
- 目录：`memory/shared/`
- 规则：每个Agent完成任务后，写关键结论到这里
- 格式：`[Agent名]_[日期]_[任务].md`

### 3. 会话历史 (方案3)
- 用 `sessions_history` 获取其他Agent最近对话
- 派发任务时作为上下文一起发送

### 4. 质量把控
- 简单任务：Agent自检后直接交付
- 复杂任务：增设二次审查

### 5. 每周复盘
- 周五输出一份「团队最佳实践」
- 更新本协作公约

## 验收标准（每个任务必带）
1. 目标是什么
2. 完成标准是什么
3. 边界在哪里

---

**记住：没有验证=没完成**
