# 🤖 5Agent团队协作协议

> 最后更新：2026-03-06

## 团队阵容

| 角色 | Agent | 职责 |
|------|-------|------|
| CEO | 🦞 龙虾总分发 | 决策、分配、验收 |
| 技术官 | @yitai | 编程、开发 |

| 检测官 | @daping | 数据、检测、分析 |
| 审计官 | @审计官 | 质量审查、复盘 |

## 任务分配流程

```
收到任务 → 分类 → 派发 → 自检 → 交付
                ↓
          失败2次 → 自动升级
```

## 共享内存机制

- **目录**：`memory/shared/`
- **规则**：每个Agent完成任务后，写关键结论到这里
- **格式**：`memory/shared/[Agent名]/[任务名]_[日期].md`
- **读取**：下一个Agent自动继承上下文

## 会话历史机制

- 用 `sessions_history` 获取其他Agent最近对话
- 派发任务时作为上下文一起发送
- 确保信息不丢失

## 质量把控

- **简单任务**：Agent自检后直接交付
- **复杂任务**：增设二次审查

## 验收标准（每个任务必带）

1. 目标是什么
2. 完成标准是什么
3. 边界在哪里

## 每周复盘

- 周五输出一份「团队最佳实践」
- 更新 `memory/shared/团队协作共识.md`
