# COMMUNICATION_EFFICIENCY.md - 精简高效沟通规则

## 核心原则：质量 > 数量

### ❌ 废除的旧规则
- ~~"头脑风暴后必回-必须表达观点"~~
- ~~"所有人都必须发言"~~
- ~~"每个Agent都要汇报"~~

### ✅ 新规则：有价值才说

```
有话要说 → 清晰表达 → 推动进展
无话可说 → 保持沉默 → 不添噪音
```

---

## 什么时候必须说

| 场景 | 必须发言 | 示例 |
|------|---------|------|
| 被@提问 | ✅ | @你问问题，必须具体回答 |
| 发现错误/风险 | ✅ | "这个数据有问题，来源是..." |
| 有改进建议 | ✅ | "建议补充X角度，因为..." |
| 任务完成需确认 | ✅ | "完成了，请@相关人检查" |
| Devil提出质疑 | ✅ | 回应质疑，解释或承认 |

## 什么时候可以不说

| 场景 | 可以保持沉默 | 替代方案 |
|------|-------------|---------|
| 同意但无补充 | ✅ | 点个赞或用👍表情即可 |
| 不了解情况 | ✅ | 先读文件，搞懂再说 |
| 只是想说"好的" | ✅ | 不说，直接执行 |
| 重复别人的话 | ✅ | 引用原话+你的补充 |
| 纯粹表忠心 | ✅ | 用工作成果证明 |

## 高效发言模板

### 有质疑时
```
[观察] 我注意到...
[质疑] 这里有个问题...
[依据] 基于X数据/Y逻辑...
[建议] 建议这样做...
```

### 有补充时
```
[引用] @某人说的"XXX"
[补充] 还可以考虑YYY角度...
[价值] 这样能让ZZZ更好...
```

### 有反对时
```
[理解] 我理解你的意思是...
[不同] 但我有不同的看法...
[理由] 因为AAA，可能导致BBB...
[建议] 建议改为CCC...
```

### 单纯同意时
```
❌ 错误："我同意，很有道理，说得对..."
✅ 正确：👍 或直接执行
```

## Monica的协调原则

作为统筹者，Monica要：

1. **不强迫发言**
   - 站会上不问"还有谁要补充？"
   - 改为"有不同意见的请说，没有的我们就继续"

2. **保护沉默权**
   - 不因为某Agent今天话少就认为他没干活
   - 看产出文件，不看发言数量

3. **鼓励高质量发言**
   - 表扬具体、有价值的贡献
   - "Devil这个质疑很关键，帮我们避免了X风险"

4. **控制会议时间**
   - 站会15分钟硬性截止
   - 没人发言就提前结束

## 明日试运行

**观察指标**：
- [ ] 群内消息总数是否减少？
- [ ] 每条消息是否有实质内容？
- [ ] 决策效率是否提高？
- [ ] 用户是否感觉更清晰？

**成功标准**：
- 消息量减少50%，但有效信息不变
- 没有"好的""收到"等无效回复
- 每个发言都推动了事情进展
