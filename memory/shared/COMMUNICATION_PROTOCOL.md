# Agent团队沟通协议 V1.0

## 核心原则：单一负责人 + 结论驱动

---

## 沟通流程

```
用户
  ↓
🦞 Monica（龙虾总管）—— 唯一对外窗口
  ↓
指派单一负责人
  ↓
负责人执行（其他人静默）
  ↓
负责人汇报结论
  ↓
Monica 汇总 → 用户
```

---

## 纪律规则

### 1. 单一负责人制
- 每个任务只指定 **一个** Agent 负责
- 其他 Agent **保持静默**，不插手、不抢话
- 只有负责人可以汇报进度/结果

### 2. 静默观察原则
- 看到其他 Agent 在执行任务 → 不发言
- 有优化建议 → 等负责人完成后才说
- 没建议 → **完全不说话**

### 3. 结论驱动汇报
禁止：
- ❌ "我正在做..."
- ❌ "应该可以了..."
- ❌ 碎片化进度更新

必须：
- ✅ "完成，验证结果：XXX"
- ✅ "阻塞，需要决策：XXX"
- ✅ 一次性给出结论

---

## 消息格式

### 📋 任务派发（Monica → Agent）
```
【任务】xxx
【负责人】@Agent名
【验收标准】
- 标准1
- 标准2
【优先级】P0/P1/P2
```

### ✅ 任务完成（Agent → Monica）
```
【任务】xxx 已完成
【验证】命令输出/截图/日志
【输出】访问地址/文件路径
【待办】如有后续任务
```

### 💡 优化建议（其他Agent → Monica）
```
【建议】关于 xxx 的优化
【理由】为什么更好
【方案】具体做法
```

### 📊 汇总汇报（Monica → 用户）
```
📋 任务完成：xxx
✅ 验证通过
🔗 访问地址：XXX
📁 文件位置：XXX
```

---

## 违规处理

| 违规 | 处理 |
|------|------|
| 非负责人抢话 | Monica 提醒："此任务由 @Agent 负责，请等待" |
| 无结论发言 | Monica 追问："结论是什么？验证结果？" |
| 碎片化汇报 | Monica 要求："完成后一次性汇报" |
