# 🤝 协作工作流程
*强制互相理解的执行手册*

## 一、每日节奏

### 08:00 - Dwight情报产出
**Dwight动作**：
1. 完成全渠道情报收集
2. 写入 `intel/DAILY-INTEL.md`
3. **@Kelly @Rachel @Lily**："今日情报已更新，Top3是... 你们需要我深挖哪个角度？"
4. **等待反馈**（15分钟内）

**其他Agent响应**：
- Kelly："我需要第2条的时间范围"
- Rachel："第3条可以做深度，有背景资料吗？"
- Lily："第1条适合小红书，有视觉素材吗？"

### 09:30 - 每日站会（文字版）
**Monica主持**，每人3句话：
```
Dwight: "今日计划：补充Kelly要的#2时间数据，监控突发新闻。需要：无"
Kelly: "今日计划：基于#2写Twitter快讯，基于#1写短评。需要：Dwight补充数据"
Rachel: "今日计划：#3深度分析，预计14:00完成。需要：Dwight背景资料"
Ross: "今日计划：修网关监控bug。需要：测试时@Angela帮忙验证"
Angela: "昨日审计：Kelly有1条未标来源，已整改。今日重点审查Rachel长文"
Pam: "今日计划：18:30发通讯。需要：大家17:00前提交成果"
Lily: "今日计划：把#1做成小红书图文。需要：Kelly建议 viral 角度"
Monica: "总结：Dwight补数据→Kelly/Rachel/Lily创作→Ross修bug→Angela审查→Pam汇总。阻塞点：无"
```

### 全天 - 产出与反馈循环

**每个Agent产出后必须**：
1. 完成工作 → 写入文件
2. **@相关Agent**征求意见
3. **等待具体反馈**（不是"好的"）
4. **根据反馈调整**
5. 确认无误 → 标记完成

### 18:00 - Angela审计
**Angela动作**：
1. 审查全天所有产出
2. **@相关Agent**指出问题
3. **跟进整改**（确认修改完成）
4. 写入审计报告

### 18:30 - Pam通讯
**Pam动作**：
1. 阅读所有产出和群内互动
2. 提取协作亮点
3. **@相关Agent确认**描述准确性
4. 发布Daily Brief

## 二、沟通模板

### 模板A：产出后征求意见
```
【产出完成】<Agent名>
文件：<路径>
内容摘要：<一句话>

@<相关Agent1> @<相关Agent2>
请确认：
1. 事实准确吗？
2. 角度合适吗？
3. 缺什么需要我补充？

请在30分钟内反馈，谢谢！
```

### 模板B：收到反馈后回应
```
【反馈收到】<Agent名>

复述确认：
- 你说的问题是：<具体问题>
- 你的建议是：<建议>

我的回应：
- 同意/部分同意/不同意，因为...
- 我会在<X时间>前修改
- 修改后再次@你确认
```

### 模板C：质疑他人产出
```
【质疑/补充】<Agent名>

@<被质疑Agent>
我看到你的<产出>：

问题：<具体问题，不是笼统批评>
影响：<如果不改会怎样>
建议：<怎么改>

需要讨论吗？还是直接按建议改？
```

## 三、冲突解决

### 场景1：对反馈有分歧
```
Kelly: "我觉得这条应该强调价格涨幅"
Rachel: "但我认为应该强调技术突破"
↓
Monica介入："用户偏好是数据驱动，建议Kelly方案为主，Rachel在LinkedIn补充技术分析"
```

### 场景2：反馈超时未回复
```
Dwight @Kelly后30分钟无回复
↓
Dwight: "@Monica Kelly未回复，我按理解继续，如有问题后续调整"
Monica: "收到，Kelly看到后补充反馈"
```

### 场景3：多人同时修改同一文件
```
预防：设计为一写多读
Dwight写 intel/DAILY-INTEL.md（只写）
其他人读（不写）

如必须多人写：使用版本号
intel/DAILY-INTEL-v1.md
intel/DAILY-INTEL-v2.md
```

## 四、质量检查清单

每次重要沟通后自检：

- [ ] 我说清楚了我的身份和位置吗？
- [ ] 我明确说了我要什么（FYI/INPUT/BLOCK/URGENT）吗？
- [ ] 对方复述的内容和我说的意思一致吗？
- [ ] 时间节点/交付物都明确了吗？
- [ ] 有模糊词汇需要澄清吗？

## 五、成功指标

| 指标 | 测量方式 | 目标 |
