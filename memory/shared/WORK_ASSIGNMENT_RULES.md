# WORK_ASSIGNMENT_RULES.md - 任务分配与检查规则

## 核心原则：不重复，只检查

### ❌ 禁止行为
- Dwight写完情报 → Kelly又自己去搜一遍
- Ross修好代码 → Angela又重新写一遍
- Pam发完简报 → Monica又汇总一次

### ✅ 正确流程
```
Agent A完成任务 → 写入共享文件 → @相关Agent检查
Agent B/C/D → 读取A的产出 → 给出改进建议 → 不重复执行
```

---

## 各任务唯一执行者

| 任务 | 唯一执行者 | 检查者 | 检查内容 |
|------|-----------|--------|---------|
| 每日情报收集 | **Dwight** | Kelly, Rachel, Angela | 完整性、准确性、角度是否够 |
| Twitter推文 | **Kelly** | Dwight(事实), Angela(质量) | 事实准确、风格合规 |
| LinkedIn长文 | **Rachel** | Dwight(事实), Kelly(角度) | 深度足够、不与Twitter冲突 |
| 工程/代码 | **Ross** | Angela(代码审查) | 安全性、规范性、可维护性 |
| 质量审计 | **Angela** | Monica(确认严重性) | 客观公正、问题分级准确 |
| 每日简报 | **Pam** | Monica(整体把关) | 完整度、重点突出 |
| 统筹协调 | **Monica** | 用户(最终确认) | 决策合理性、资源分配 |

---

## 检查反馈模板（必须用）

```
[检查] @执行者 的"任务名"
[状态] ✅通过 / ⚠️需修改 / ❌有问题
[优点] 做得好的地方...
[建议] 具体改进建议...
[可选] 我可以补充的是...
[时间] 建议XX时间内完成修改
```

**示例**：
```
[检查] @Dwight 的"今日情报"
[状态] ⚠️需补充
[优点] Top3选得准，BTC突破抓得好
[建议] 第2条AI框架缺中文社区反应
[可选] 我可以用Nitter补一下中文讨论
[时间] 30分钟内补充即可
```

---

## 发现重复工作的处理

如果Angela发现Kelly又在搜情报：
```
[提醒] @Kelly Dwight已经在08:00完成了情报收集
[建议] 你直接读intel/DAILY-INTEL.md即可
[行动] 如需补充角度，请@Dwight协作，不要重复采集
[记录] 记入angela/duplication-log.md
```

---

## 例外情况

可以重复的情况：
1. **验证性质** - Angela验证数据准确性（不是重新采集）
2. **补充性质** - Kelly发现Dwight缺中文源，补充而非替代
3. **用户明确要求** - "Dwight的情报不够，Kelly你再搜一下"

需要Monica批准才能重复。
