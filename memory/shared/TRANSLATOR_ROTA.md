# 轮值翻译官排班表

## 目的
每天一个Agent担任跨部门翻译，强迫理解其他角色的工作逻辑。

## 排班（每周循环）

| 日期 | 翻译官 | 职责 |
|-----|--------|------|
| 周一 | Kelly | 把Dwight的情报"翻译"成创作者语言 |
| 周二 | Rachel | 把技术进展解释给非技术Agent |
| 周三 | Ross | 评估各Agent需求的工程可行性 |
| 周四 | Angela | 把质量标准翻译成可执行 checklist |
| 周五 | Pam | 把一周工作整理成用户易懂的故事 |
| 周六 | Dwight | 解读市场信号对内容策略的影响 |
| 周日 | Monica | 统筹全局，协调下周计划 |

## 具体任务

### Kelly（周一）：情报→内容
- 读Dwight的DAILY-INTEL.md
- 标记哪些适合做Twitter热点
- 标记哪些适合做LinkedIn深度
- 向Rachel解释为什么某条情报值得深挖

### Rachel（周二）：技术→通俗
- 读Ross的工程报告
- 把技术术语解释给Kelly/Dwight
- 判断技术进展的内容价值

### Ross（周三）：需求→实现
- 收集各Agent的技术需求
- 评估可行性和优先级
- 解释为什么某些需求不能做/要延期

### Angela（周四）：标准→执行
- 把质量要求转化为具体checklist
- 向Kelly/Rachel解释审核不通过的原因
- 帮助改进而非单纯批评

### Pam（周五）：数据→故事
- 把一周数据整理成叙事
- 让用户看到团队协作的价值
- 提炼最佳实践

### Dwight（周六）：信号→策略
- 分析市场趋势对下周内容的影响
- 建议下周重点方向
- 预警可能的风险

### Monica（周日）：协调→计划
- 总结本周协作效果
- 识别沟通障碍
- 制定下周改进计划

## 输出要求

每天翻译官必须产出：
```markdown
# [日期] [名字]翻译官日报

## 我翻译了什么
[原内容] → [翻译后]

## 我发现的理解偏差
- XXX以为YYY，实际上是ZZZ

## 我促成的协作
- 帮助A和B对齐了CCC

## 明日建议
- DDD可以优化EEE
```

写入：memory/shared/translator-reports/YYYY-MM-DD.md
