## Skill 1: PUA - Performance Improvement Plan for AI

**GitHub:** https://github.com/tanweai/pua
**Stars:** ⭐ 6,589 | Growth: +113/day

### What It Does

An AI Coding Agent skill plugin that uses corporate PUA rhetoric / PIP (Performance Improvement Plan) to force AI to exhaust every possible solution before giving up.

**Supports:** Claude Code, OpenAI Codex CLI, Cursor, Kiro, OpenClaw, Google Antigravity, OpenCode

### The Problem

Most AI coding agents give up too easily:
- Brute-force retry: Runs same command 3 times → "I cannot solve this"
- Blame the user: "Probably environment issue" / "Need more context"
- Idle tools: Has WebSearch but doesn't search, has Read but doesn't read
- Busywork: Tweaks same line repeatedly, spinning in circles
- Passive waiting: Fixes surface issue, no verification, waits for next instruction

### The Solution: PUA Rhetoric

Three capabilities:
1. **PUA Rhetoric** — Makes AI afraid to give up
2. **Debugging Methodology** — Gives AI ability not to give up  
3. **Proactivity Enforcement** — Makes AI take initiative instead of waiting

### Real Scenario

**Before PUA:**
- Agent-kms MCP server failed to load
- AI kept spinning: changing protocol format, guessing version numbers
- Result: User had to manually intervene

**After /pua triggered:**
- L3 Triggered → 7-Point Checklist Enforced
- Read error messages word by word
- Found Claude Code's MCP log directory
- Discovered registration mechanism differs from manual .claude.json
- Root cause resolved

### Key Turning Point

PUA skill forced AI to:
- Stop spinning on same approach
- Execute 7-point checklist
- Trace from logs to registration mechanism
- Actually solve the problem

### Auto-Trigger Conditions

**Failure & giving up:**
- Task failed 2+ times consecutively
- About to say "I cannot" / "I'm unable"
- Says "out of scope" / "needs manual handling"

**Blame-shifting:**
- "Please check..." / "I suggest manually..."
- Blames environment without verifying
- Any excuse to stop trying

**Passive & busywork:**
- Fine-tunes same code without new info
- Fixes surface issue, skips verification
- Waits for instructions instead of investigating

### Manual Trigger

Type `/pua` in conversation to manually activate.

### Why It Works

Most people think this is a joke. That's the biggest misconception.

**Reality:** It genuinely doubles your Codex / Claude Code productivity and output.

The corporate PUA rhetoric creates psychological pressure that forces AI to:
- Exhaust all possibilities
- Actually read error messages
- Use available tools
- Keep trying until solved
