# 🎮 群内指挥协议
*解决多人同时说话的混乱*

## 核心原则：分层汇报制

### 第一层：Agent → Monica（龙虾总管）
所有Agent只向Monica汇报，不直接@用户

### 第二层：Monica → 用户
Monica汇总后，统一向用户汇报

---

## 📋 标准化汇报模板

### Agent完成工作时发送：
```
【完成】<任务名>
负责人：<Agent名>
结果：<一句话总结>
详情：<文件路径>
状态：✅ 已完成 / ⏳ 待审核
```

### Monica汇总时发送：
```
📊 <时间> 工作汇总

✅ 已完成：
- Dwight：收集XX条情报
- Ross：修复XX问题
- ...

⏳ 待你确认：
- Rachel：LinkedIn草稿X篇
- Lily：小红书笔记X篇
请回复"批准"或给出修改意见
```

---

## 🚫 禁止行为

❌ Agent直接@用户
❌ 多个Agent同时汇报
❌ 长篇大论无重点
❌ 重复发送相同内容

---

## ✅ 正确示例

### 场景1：Dwight完成情报收集
**Dwight发送**：
```
【完成】08:00情报收集
负责人：Dwight
结果：收集47条，Top3已标注
详情：intel/DAILY-INTEL.md
状态：✅ 已完成
```

**Monica汇总后发送**：
```
📊 08:30 早间工作汇总

✅ 已完成：
- Dwight：情报收集47条（Crypto:15, AI:20, Tech:12）

⏳ 待处理：
- Rachel：等待读取情报生成LinkedIn
- Lily：等待读取情报生成小红书

📎 附件：intel/DAILY-INTEL.md
```

### 场景2：需要用户决策时
**Monica发送**：
```
📝 需要你确认

Rachel生成了3篇LinkedIn草稿：
1. 《AI Agent趋势分析》- 专业深度 ✅推荐
2. 《本周Crypto热点》- 时效性强 ✅推荐  
3. 《技术工具分享》- 教育价值 ✅推荐

Lily生成了3篇小红书笔记：
1. 「3个AI神器」- 爆款潜力 🔥
2. 「程序员效率工具」- 垂直精准 🎯
3. 「AI入门指南」- 新手友好 📚

回复"全部批准"或编号如"批准LinkedIn-1,2 小红书-1"
```

## 🆘 紧急情况处理

如果Agent遇到紧急问题：
```
🚨 紧急
Agent：<名字>
问题：<一句话描述>
需要：<具体帮助>
```

Monica立即处理并@用户

---

## 📱 用户指令格式

你可以这样下指令：

```
"批准今日所有内容" → Monica转发给各Agent执行

"查看今日情报" → Monica发送DAILY-INTEL摘要

"暂停小红书" → Monica通知Lily暂停任务

"紧急：修复网关" → Monica立即派Ross处理
```
