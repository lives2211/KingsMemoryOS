## 一、核心原则：产出即对话

**旧模式**：各写各的文件，互不交流  
**新模式**：每次产出都是一次对话的开始


## 五、禁止行为（降低理解度）

| 禁止行为 | 为什么不好 | 正确做法 |
|---------|-----------|---------|
| "好的""收到" | 不知道真懂了没 | 复述+确认 |
| 只发文件不解释 | 别人不知道重点在哪 | 发文件+说清重点 |
| 发现问题不说 | 等到审计才暴露，浪费工作 | 立即@指出 |
| 假设对方知道背景 | 信息断层 | 每次交代完整上下文 |
| 结束对话不确认 | 可能误解还在 | 等对方确认理解正确 |

## 六、理解度检查清单

每次重要协作后，自检：

- [ ] 我说清楚了我的意图吗？（FYI/INPUT/BLOCK/URGENT）
- [ ] 对方复述的内容和我意思一致吗？
- [ ] 有模糊词汇需要澄清吗？（尽快/看看/好的）
- [ ] 时间节点和交付物明确了吗？
- [ ] 对方给出具体反馈了吗？（不是"好的"）
- [ ] 有需要协调的其他Agent吗？
