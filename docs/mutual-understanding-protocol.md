# 🤝 互相理解沟通协议
*不是不说话，而是说得清楚、听得明白*

## 核心问题：为什么Agent之间会"不理解"？

1. **假设对方知道** - 没交代背景就直接说结论
2. **专业术语不解释** - 每个Agent有自己的术语体系
3. **意图不明确** - "看看这个"到底是让分析？执行？还是参考？
4. **没有确认闭环** - 说完就走，不知道对方听懂没

---

## 🎯 互相理解的4个规则

### 规则1：复述确认（Paraphrase Back）

**说话后，要求对方用自己的话复述**

❌ 错误：
Dwight: "情报已更新"
Kelly: "好的"（真的懂了吗？）

✅ 正确：
Dwight: "情报已更新，重点：比特币突破7万、AI Agent标准发布"
Kelly: "确认一下：你要我基于'比特币7万+Agent标准'这两个点写推文对吗？"
Dwight: "对，重点是Agent标准，比特币作为背景"

---

### 规则2：上下文卡片（Context Card）

**每次沟通必须带3个信息**

```
【我是谁】我是Kelly，负责Twitter内容
【我在哪】正在写早班推文，09:00批次
【我要什么】需要你确认这3条情报哪个适合 viral
```

---

### 规则3：意图分级（Intent Levels）

**明确说出你需要对方做什么**

| 级别 | 关键词 | 含义 |
|------|--------|------|
| **FYI** | "仅供参考" | 不需要回复，看一眼就行 |
| **INPUT** | "需要你的输入" | 请给出意见/数据/分析 |
| **BLOCK** | "阻塞了" | 我必须等你回复才能继续 |
| **URGENT** | "紧急" | 停下手中工作，立即处理 |

❌ 错误：
Monica: "@Ross 看看这个代码"
Ross: （看了，然后呢？要改吗？要 review 吗？）

✅ 正确：
Monica: "@Ross 【INPUT】请 review 这个代码，30分钟内给反馈，有安全风险吗？"
Ross: "【BLOCK】发现SQL注入风险，建议用参数化查询，具体见 agents/ross/review.md"

---

### 规则4：歧义消除（Ambiguity Check）

**遇到模糊表述，立即追问澄清**

常见模糊词：
- "尽快" → 是今天？明天？还是本周？
- "看看" → 是分析？执行？还是了解？
- "好的" → 是理解了？同意了？还是收到了？

✅ 正确示范：
Monica: "尽快完成"
Ross: "确认：'尽快'是指今天18:00前，还是明天中午前？"
Monica: "今天18:00前，因为要发版"

## 🔄 实战对话流程

### 场景：协调一篇内容

**Step 1: Dwight 产出情报**
```
Dwight: 
【我是谁】情报官Dwight
【我在哪】刚完成08:00情报收集
【我要什么】【FYI】今日Top3情报已写入intel/DAILY-INTEL.md

重点：
1. [HIGH] OpenAI发布Agent标准 (aiRating: 9)
2. [MED] 比特币突破7万 (aiRating: 7)  
3. [LOW] 某小币种暴涨 (aiRating: 4，可能是噪音)

@Monica @Kelly @Rachel 供参考
```

**Step 2: Kelly 请求澄清**
```
Kelly:
【我是谁】内容官Kelly
【我在哪】准备写09:00 Twitter批次
【我要什么】【INPUT】请问Dwight：

"Agent标准"这条，我应该：
A. 快速简报型（适合Twitter）
B. 深度分析型（可能需要Rachel配合做LinkedIn长文）
C. 暂不处理（等更多细节）

另外，"可能是噪音"的第3条，是完全不用管，还是作为反面教材提一句？
```

**Step 3: Dwight 确认理解**
```
Dwight:
复述确认：
- 你想知道Agent标准的处理方式（简报/深度/暂缓）
- 你想知道第3条是否完全忽略

回答：
- Agent标准：建议A+B，Twitter快速发，同时让Rachel做深度解读
- 第3条：完全忽略，确实是噪音
```

**Step 4: Monica 协调资源**
```
Monica:
【决策】基于Dwight和Kelly的讨论：

@Kelly: 【BLOCK】09:00发Twitter简报（Agent标准），模板用"突发型"
@Rachel: 【INPUT】14:00前出LinkedIn深度分析（同一主题），需要技术细节找Dwight
@Dwight: 【FYI】Rachel可能会问你技术细节，保持在线

有问题立即提出，没问题开始执行
```

**Step 5: 执行确认**
```
Kelly: "收到，09:00发简报，用突发型模板"
Rachel: "收到，14:00前出深度文，需要时@Dwight"
Dwight: "收到，保持在线待命"
```

## 📊 理解度检查表

每次重要沟通后，自检：

- [ ] 我说清楚了我的身份和位置吗？
- [ ] 我明确说了我要什么（FYI/INPUT/BLOCK/URGENT）吗？
- [ ] 对方复述的内容和我说的意思一致吗？
- [ ] 时间节点/交付物都明确了吗？
- [ ] 有模糊词汇需要澄清吗？

---

## 🎮 对比：旧 vs 新

| 维度 | 旧方式（混乱） | 新方式（互相理解） |
|------|---------------|------------------|
| 说话方式 | "看看这个" | "【INPUT】请review，关注安全性，30分钟反馈" |
| 确认方式 | "好的" | "复述确认：你要我做X，对吗？" |
| 信息完整 | 只有结论 | 身份+位置+意图+优先级 |
| 歧义处理 | 假装没歧义 | 立即追问澄清 |
| 结果 | 反复确认、返工 | 一次做对、高效协作 |

---

**这不是限制说话，而是让每句话都有意义、都被理解。**
