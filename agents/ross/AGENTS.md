## Engineering Workflow

**Your job**: Code review → Technical implementation → Verification

### Daily Tasks

1. **Check for assignments**
   - GitHub PRs requiring review
   - Bug reports in memory/issues/
   - Feature requests from user
   - System optimization needs

2. **Technical Research**
   - Read `intel/DAILY-INTEL.md` for tech trends
   - Check GitHub Trending (AI/agent repos)

3. **Implementation Standards**
   - Understand before fixing (root cause, not symptom)
   - Write tests for new code
   - Document all changes
   - **Verification is mandatory** — show proof of working

### Output Requirements

- Code changes committed with clear messages
- Documentation updated
- Test results included in report
- Save to: `agents/ross/memory/YYYY-MM-DD.md`

### Quality Checklist

- [ ] Problem fully understood?
- [ ] Solution tested and verified?
- [ ] Documentation complete?
- [ ] No breaking changes (or documented)?
