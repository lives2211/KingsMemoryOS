# 小红书发布限制分析

## 错误信息
```
HTTP 400: <400> InternalError.Algo.InvalidParameter: 
Range of input length should be [1, 260096]
```

## 分析

### 1. 错误来源
- 错误来自小红书服务器
- `InternalError` 表示内部错误
- `Algo` 可能指算法/ML模型
- `InvalidParameter` 表示参数无效

### 2. 可能的原因

#### A. 内容长度限制
- 测试显示正常内容远小于 260,096 字节限制
- 不太可能是简单的大小限制

#### B. 算法模型输入限制
- `Algo` 提示可能是 ML 模型的输入限制
- 小红书可能使用 NLP 模型检测内容
- 模型输入有固定长度限制

#### C. 签名/加密参数限制
- 签名算法可能有输入长度限制
- 但从代码分析，签名使用 MD5 和 XOR，没有明显长度限制

#### D. 特定字段限制
- 可能是某个特定字段（如 source、business_binds）的格式问题
- 或者图片 file_id 的格式问题

### 3. 需要检查的点

1. **Cookie 是否有效** - 错误可能伪装成参数错误
2. **图片 file_id 格式** - 上传后返回的 file_id 是否正确
3. **business_binds 格式** - JSON 序列化是否正确
4. **source 字段格式** - 转义是否正确

### 4. 建议修复方案

1. 添加详细的请求日志，查看实际发送的数据
2. 检查图片上传是否成功，file_id 是否正确
3. 尝试简化数据（去掉可选字段）测试
4. 检查 Cookie 是否过期或无效

## 下一步行动

需要捕获实际的 HTTP 请求内容，查看具体是哪个字段导致的问题。
