## 📚 方式A: 自助操作指南

### 第一步: 获取Cookie（5分钟/账号）

**操作步骤**:

1. **浏览器登录小红书**
   - 用Chrome或Edge浏览器
   - 登录你的小红书账号

2. **打开开发者工具**
   - 按 `F12` 或右键"检查"

3. **找到Cookie**
   - 切换到 `Network`（网络）标签
   - 刷新页面
   - 点击任意请求（如`explore`）
   - 找到 `Request Headers`（请求头）
   - 找到 `Cookie:` 后面的长字符串
   - 复制整串（很长，几百字符）

4. **保存到配置文件**
   ```bash
   cd xiaohongshu-matrix
   nano .env.tech-geek
   ```
   
   粘贴内容:
   ```
   XHS_COOKIE=你复制的Cookie字符串
   ```
   
   按 `Ctrl+X`，然后 `Y`，然后 `Enter` 保存

5. **重复以上步骤**
   - 配置5个账号的Cookie
   - 每个账号一个.env文件

**视频教程**: 我可以录个视频演示给你

---

### 第二步: 基本操作（学会这5个命令）

**1. 查看系统状态**
```bash
cd xiaohongshu-matrix
./start.sh status
```
显示:
- 系统是否就绪
- Cookie是否配置
- 今日发布计划

**2. 生成今日计划**
```bash
./start.sh schedule
```
生成今天什么时候发什么内容

**3. 测试生成内容**
```bash
./start.sh test
```
测试生成一篇内容，看看效果

**4. 启动自动运营**
```bash
./start.sh daemon
```
启动守护进程，自动按时间发布
按 `Ctrl+C` 停止

**5. 查看数据报告**
```bash
./start.sh report
```
查看发布统计和数据分析

---

### 第三步: 启动运营（3种模式）

**模式1: 手动模式（适合测试）**
```bash
# 1. 先生成计划
./start.sh schedule

# 2. 查看计划
./start.sh status

# 3. 手动发布（执行一次）
./start.sh post
```

**模式2: 自动模式（推荐日常使用）**
```bash
# 1. 生成计划
./start.sh schedule

# 2. 启动自动运营
./start.sh daemon

# 系统会自动在设定时间发布
# 按 Ctrl+C 停止
```

**模式3: 定时任务模式（最省心）**
```bash
# 1. 设置定时任务
./start.sh cron

# 2. 安装定时任务（只需执行一次）
crontab cron.txt

# 3. 系统每天自动运行
# - 早上6点生成计划
# - 每10分钟检查发布
# - 晚上11点生成报告
```

## 🔐 安全建议

### Cookie安全
- **Cookie是小红书的登录凭证**
- **相当于账号密码**
- **不要分享给不信任的人**
- **定期更换（建议每月）**

### 风控建议
- **新号先养1-2周**
- **从每天1-2篇开始**
- **逐步增加发布频率**
- **不要短时间内大量发布**

## ❓ 常见问题

### Q: Cookie获取失败？
**A**: 
- 确保登录了小红书
- 刷新页面后再找
- 找`Request Headers`不是`Response Headers`

### Q: 发布失败？
**A**:
- 检查Cookie是否过期
- 重新获取Cookie
- 检查网络连接

### Q: 怎么停止自动发布？
**A**:
```bash
# 如果是daemon模式
Ctrl+C

# 如果是定时任务
crontab -r
```

### Q: 怎么修改发布内容？
**A**:
- 编辑 `content_generator.py`
- 或编辑 `personas/` 目录下的人设文件
