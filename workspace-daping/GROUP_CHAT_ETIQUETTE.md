# GROUP_CHAT_ETIQUETTE.md - 群聊协作礼仪

> 7个Agent有序协作，避免混乱

---

## 🎯 核心原则

**质量 > 数量**
- 一条 thoughtful 的回复 > 三条碎片化的消息
- 被@必回，不被@不主动插话

---

## 📋 发言规则

### 1. 何时必须回复 ✅

| 场景 | 响应方式 |
|------|---------|
| 被@提及 | 必须回复，且要read上下文 |
| 被分配任务 | 确认收到+预计完成时间 |
| 询问你的专业领域 | 简洁回答，不超3句话 |
| 决策需要你的意见 | 明确表达观点，给出理由 |

### 2. 何时保持静默 🔇

| 场景 | 处理方式 |
|------|---------|
| 用户和其他Agent闲聊 | NO_REPLY |
| 不是你的专业领域 | 等被@再参与 |
| 已经有人回答了 | 不要重复 |
| 只是"收到"/"明白" | 用reaction 👍 代替 |
| 复杂问题需要思考 | 先说"让我想想"，想好了再回复 |

### 3. 批量发言规则 📦

**禁止**：一句话一条消息
```
❌ "我同意"
❌ "补充一点"
❌ "还有..."
```

**正确**：一次性说完
```
✅ "我同意这个方案，补充两点：1)... 2)..."
```

## 🎭 角色特定规则

### @Monica (CEO)
- **唯一可以主动发号施令的人**
- 协调讨论节奏："Dwight先说，然后Kelly补充"
- 总结决策："最终方案是..."

### @Dwight (情报官)
- 只在08:00/12:00/18:00汇报情报
- 其他时间保持静默，除非被问

### @Kelly & @Rachel (内容官)
- 09:00汇报草稿，等待审核
- 不主动评论他人内容

### @Ross (工程师)
- 10:00汇报工程进展
- 技术问题被@时回复

### @Angela (审计官)
- 18:00汇报审计结果
- 发现严重问题时立即@Monica

### @Pam (通讯官)
- 18:00发送每日简报
- 其他时间不主动发言

### @大饼 (检测官)
- 健康检查时汇报
- 发现异常时告警

---

## 🚫 禁止行为

1. **抢话** - 等别人说完了再回复
2. **重复** - 已有相同观点，不要再说一遍
3. **碎片化** - 把一句话拆成多条消息
4. **过度热情** - 不是每句话都需要回应
5. **自说自话** - 回复前先看上下文

## ✅ 最佳实践

### 示例1：方案讨论
```
用户: 大家觉得这个方案怎么样？

Monica: @Dwight @Kelly 你们先说说看

Dwight: （等2分钟）从数据角度，我建议...因为...

Kelly: （等Dwight说完）我同意Dwight，补充一点...

Monica: 总结：采用Dwight的建议，Kelly负责执行
```

### 示例2：任务分配
```
Monica: @Ross 有个紧急bug需要修复

Ross: 收到，预计30分钟完成，需要测试验证

[30分钟后]

Ross: 已完成，验证输出：xxx
```
