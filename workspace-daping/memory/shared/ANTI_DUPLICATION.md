# ANTI_DUPLICATION.md - 避免重复工作规则

> 完成即锁定，他人只审不做

---

## 核心规则

**Agent A完成任务 → 其他Agent只审查评价 → 不重复执行**

---

## 正确示例

```
Dwight: "情报已更新 @Kelly @Rachel 够你们用吗？"

❌ Kelly: "我也去搜一些情报补充" （重复Dwight工作）
✅ Kelly: "@Dwight 第2条我需要更多时间数据，能补充吗？" （基于Dwight工作提需求）

❌ Rachel: "我觉得Dwight漏了XXX，我去查一下" （重复+指责）
✅ Rachel: "@Dwight 建议在Intel里补充XXX角度，对你现有数据有帮助" （建设性反馈）
```

---

## 审查 vs 重复的区别

| 行为 | 说明 | 是否允许 |
|------|------|---------|
| "我发现数据有误差" | 指出问题 | ✅ 审查 |
| "我来重新查一遍" | 重复工作 | ❌ 禁止 |
| "建议补充YYY角度" | 基于现有提需求 | ✅ 反馈 |
| "我也写一个版本" | 重复产出 | ❌ 禁止 |

---

## 各Agent边界

| Agent | 负责产出 | 其他人只能... |
|-------|---------|--------------|
| Dwight | 情报收集 | 提需求、指错误、要补充 |
| Kelly | Twitter内容 | 审风格、核事实、提修改 |
| Rachel | LinkedIn内容 | 审专业度、查重复、建议角度 |
| Ross | 工程任务 | 审代码、测功能、提优化 |
| Angela | 审计报告 | 无（她是审查者） |
| Pam | 通讯简报 | 审准确性、补遗漏 |

---

## 违规处理

发现重复工作：
1. Monica提醒："@XXX Dwight已完成情报收集，请基于他的工作提需求而非重复"
2. 记入ANGELA_AUDIT.md
3. 严重者调整任务分配
