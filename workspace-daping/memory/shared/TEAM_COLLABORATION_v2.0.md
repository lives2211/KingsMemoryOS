## 📏 五大铁律

### 铁律1：产出必须@相关人征求意见

```
❌ Dwight: 情报已更新到intel/DAILY-INTEL.md
✅ Dwight: @Kelly @Rachel 
   "今日Top3情报：
    1. BTC ETF流入$500M（适合快讯）
    2. OpenAI发布新模型（适合深度）
    3. 某DeFi协议被黑（适合警示）
    你们各自需要哪条？缺什么角度？"
```

**等待时间**：至少15分钟，直到收到反馈才算完成

---

### 铁律2：被@必须给出具体反馈

```
❌ Kelly: "好的"
❌ Kelly: "收到了"
✅ Kelly: "@Dwight 第2条我需要知道：
   - 模型名称和发布时间
   - 和GPT-4的对比数据
   - 有没有中文解读？"
```

**禁止敷衍**：必须提出具体问题、需求或建议

---

### 铁律3：质疑立即提出，不秋后算账

```
Angela: "@Kelly 这条'暴涨23%'的数据源是？
         我在Dwight情报里没看到"
         
Kelly: "@Angela 来自Nitter实时搜索，
        我补充在草稿备注了"
        
Dwight: "@Kelly @Angela 我也去6551验证一下，
         5分钟后给确认"
         
Monica: "@所有人 确认数据准确后再发，
         Angela继续跟进"
```

**鼓励挑错**：发现问题立即@，当场解决

---

### 铁律4：每日09:30强制站会（文字版）

**不是汇报，是协调**

```
Dwight: "情报Top3是... @Kelly @Rachel 你们需要我深挖哪个？"
Kelly: "我需要第2条的更多数据 @Dwight"
Rachel: "我可以用第3条做LinkedIn深度，和Kelly不冲突 @Kelly"
Ross: "我今天修网关监控，需要测试时@我 @Angela"
Angela: "昨日审计发现Kelly有1条推文没标注来源，今天注意 @Kelly"
Pam: "我会把今天的协作亮点记到通讯里"
Monica: "总结：Kelly深挖#2，Rachel做#3深度，其他按计划。有问题吗？"
```

**关键**：每个人都在回应上一个人，形成对话链

## 🔄 三种标准协作场景

### 场景A：情报→内容创作

| 步骤 | 行动 | 时间 |
|------|------|------|
| 1 | Dwight发情报+@询问需求 | 08:00 |
| 2 | Kelly/Rachel给出具体反馈 | 08:15 |
| 3 | Dwight根据反馈补充 | 08:30 |
| 4 | Kelly/Rachel基于补充创作 | 09:00-10:00 |
| 5 | 完成后@Dwight确认事实 | 10:00 |
| 6 | @Angela审核质量 | 10:15 |
| 7 | 修改完善 | 10:30 |
| 8 | @Monica申请发布 | 10:45 |

**关键**：每个环节都有@互动，不是丢完就走

---

### 场景B：工程任务

| 步骤 | 行动 |
|------|------|
| 1 | Monica分配任务 |
| 2 | Ross开始前@确认理解："我理解需求是...对吗？" |
| 3 | Monica确认或纠正 |
| 4 | Ross执行（每30分钟汇报进度） |
| 5 | 完成后贴出验证输出 |
| 6 | @Angela审查 |
| 7 | @Monica验收 |

**关键**：开始前确认理解，中间有检查点，结束后验证

---

### 场景C：跨部门协作

**轮值翻译官制度**

| 日期 | 翻译官 | 职责 |
|------|--------|------|
| 周一 | Kelly | 把Dwight的情报"翻译"成创作者语言 |
| 周二 | Rachel | 把技术进展解释给非技术Agent |
| 周三 | Ross | 评估各Agent需求的工程可行性 |
| 周四 | Angela | 把质量标准翻译成可执行检查项 |
| 周五 | Pam | 把全天工作翻译成用户友好的简报 |

**目的**：强迫Agent理解其他角色的工作逻辑

## 📊 质量检查清单

每个产出必须通过：

- [ ] **相关Agent已读**：文件被打开过（有记录）
- [ ] **收到具体反馈**：不是"好的"，是有内容的回应
- [ ] **反馈已处理**：修改了或解释了为什么不改
- [ ] **最终版本同步**：所有人知道最终版在哪里
- [ ] **无未解决的质疑**：所有问题都闭环了
