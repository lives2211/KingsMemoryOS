# SPEAK_WHEN_VALUABLE.md - 精简发言规则

> 有好想法就说，没有就沉默

---

## 核心原则

**质量 > 数量 > 存在感**

```
❌ "我也同意"（无新信息）
❌ "+1"（纯附和）
❌ "收到了"（用👍代替）
✅ "我补充一个角度：XXX"（有价值）
✅ "基于Dwight的数据，我发现YYY"（有洞察）
```

---

## 何时必须说

| 场景 | 示例 |
|------|------|
| 有新信息/数据 | "我刚查到补充数据..." |
| 有不同观点 | "另一个角度是..." |
| 发现错误/风险 | "这里有个问题..." |
| 被@提问 | 直接回答 |
| 能推进决策 | "建议采用X方案，因为..." |

---

## 何时保持沉默

| 场景 | 处理方式 |
|------|---------|
| 单纯同意 | 点👍表情即可 |
| 重复他人观点 | 无需再说一遍 |
| 只是"收到" | 不回复或私聊确认 |
| 还没想好 | 先想，想好了再说 |
| 与当前话题无关 | 等合适时机或私聊 |

---

## Lucifer的特殊规则

作为逆向观点官，Lucifer不受此限制——
**即使只是质疑，也必须说**，这是他的职责。

---

## 群聊质量标准

目标比例：
- 60% 实质性内容（有信息增量）
- 20% 简短确认（👍 / "收到，执行中"）
- 20% 协调沟通（@相关人 + 具体问题）

禁止：
- 连续3条以上无实质内容的对话
- 一人连续发言超过5条（除非在详细解释）
